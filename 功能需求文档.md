## 项目概述

CPA评价管理系统是一个面向餐饮企业的评价管理和AI分析平台，包含：
- **管理后台**: 系统级管理功能（HQ/运营商视图）
- **移动端**: 商户视图功能（单店管理）

### 技术栈
- **前端**: React + TypeScript + Vite + Tailwind CSS
- **后端（待开发）**: Python + FastAPI + SQLAlchemy + PostgreSQL
- **AI模块**: 多LLM提供商集成 + 智谱AI视觉
- **RPA工具**: Playwright浏览器自动化（已独立开源）

### 1.4 AI配置 (AIConfig)

**文件路径**: `src/pages/admin/AIConfig.tsx` 及子页面

#### 功能描述
包含5个子模块：
1. **模型配置 (ModelConfig)**: 管理多平台大模型接口，进行并发质量测试
2. **深度指令 (PromptConfig)**: 配置AI角色人格、语气、字数及词库过滤规则
3. **规则引擎 (RuleEngine)**: 编排复杂的处理流水线，设置条件触发与冲突优先级
4. **实时监控 (Monitoring)**: 实时监控AI处理链路日志，进行A/B测试实验
5. **效能评估 (Evaluation)**: 基于真实反馈优化AI表现，建立持续迭代的反馈闭环

#### API接口需求
| 接口 | 方法 | 路径 | 描述 |
|------|------|------|------|
| 获取AI配置 | GET | `/api/v1/ai/config` | 返回所有AI配置 |
| 更新模型配置 | PUT | `/api/v1/ai/models` | 更新模型配置 |
| 测试模型连接 | POST | `/api/v1/ai/models/test` | 测试模型连接 |
| 更新提示词配置 | PUT | `/api/v1/ai/prompts` | 更新提示词配置 |
| 更新规则引擎 | PUT | `/api/v1/ai/rules` | 更新规则引擎配置 |
| 获取实时监控数据 | GET | `/api/v1/ai/monitoring` | 返回实时监控数据 |
| 创建A/B测试 | POST | `/api/v1/ai/ab-test` | 创建A/B测试实验 |
| 获取效能评估 | GET | `/api/v1/ai/evaluation` | 返回效能评估数据 |
| 导出AI配置 | GET | `/api/v1/ai/export` | 导出AI配置文件 |
| 同步配置到节点 | POST | `/api/v1/ai/sync` | 同步配置到所有节点 |

#### 数据库表设计
```sql
CREATE TABLE ai_model_configs (
  id SERIAL PRIMARY KEY,
  provider VARCHAR(50) NOT NULL, -- 'openai', 'zhipu', 'wenxin', etc.

  is_active BOOLEAN DEFAULT TRUE,

  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

CREATE TABLE ai_prompt_templates (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  template_text TEXT NOT NULL,
  variables TEXT[], -- 变量列表
  is_default BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE ai_rule_engines (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  description TEXT,
  rules JSONB NOT NULL, -- 规则配置（JSON格式）
  priority INTEGER DEFAULT 0,
  is_active BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE ai_processing_logs (
  id SERIAL PRIMARY KEY,
  review_id INTEGER REFERENCES reviews(id),
  model_used VARCHAR(100),
  prompt_template_id INTEGER REFERENCES ai_prompt_templates(id),
  input_text TEXT,
  output_text TEXT,
  processing_time DECIMAL(10,3), -- 秒
  status VARCHAR(20), -- 'success', 'failed'
  error_message TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

```

## 数据库设计建议

### 核心表关系图

```
users (用户/管理员)
  ↓
stores (门店) ← store_platforms (门店平台关联)
  ↓
reviews (评价) ← reply_audits (回复审核)
  ↓
ai_processing_logs (AI处理日志)

spider_platforms (爬虫平台) ← spider_sync_logs (同步日志)

notification_channels (通知渠道) ← notification_rules (通知规则)
  ↓
notification_history (推送历史)

ai_model_configs (AI模型配置)
ai_prompt_templates (提示词模板)
ai_rule_engines (规则引擎)

roles (角色) ← users.role_id
organization_structure (组织架构) ← organization_members

competitors (竞品) ← competitor_analysis_tasks (分析任务)

annual_reports (年度报告)
```

### 数据库选型建议

**主数据库**: PostgreSQL 14+
- 支持JSONB类型（用于存储灵活配置）
- 支持数组类型（用于存储标签、图片等）
- 强大的查询性能

**缓存**: Redis
- 缓存热点数据（统计数据、配置信息）
- Session存储
- API限流

**向量数据库**（可选，用于AI功能）: pgvector (PostgreSQL扩展)
- 存储评价向量
- 支持语义搜索

### 索引建议

```sql
-- reviews表索引
CREATE INDEX idx_reviews_store_id ON reviews(store_id);
CREATE INDEX idx_reviews_sentiment ON reviews(sentiment);
CREATE INDEX idx_reviews_platform ON reviews(platform);
CREATE INDEX idx_reviews_review_time ON reviews(review_time);
CREATE INDEX idx_reviews_content_gin ON reviews USING GIN(to_tsvector('chinese', content));

-- stores表索引
CREATE INDEX idx_stores_status ON stores(status);
CREATE INDEX idx_stores_type ON stores(type);

-- ai_processing_logs表索引
CREATE INDEX idx_ai_logs_review_id ON ai_processing_logs(review_id);
CREATE INDEX idx_ai_logs_created_at ON ai_processing_logs(created_at);
```
